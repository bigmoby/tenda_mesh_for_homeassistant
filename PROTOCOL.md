### Password Hash (encryptAccountPassword)
Algorithm extracted from `Utils.encryptAccountPassword(username, password)`:

```python
import hashlib

def encrypt_account_password(username: str, password: str) -> str:
    # Step 1: SHA1 of the username (email or phone)
    sha1 = hashlib.sha1()
    sha1.update(username.encode('utf-8'))
    sha1_username = sha1.digest()  # 20 bytes

    # Step 2: password bytes padded to multiple of 4
    pwd_bytes = password.encode('utf-8')
    align_len = ((len(pwd_bytes) + 3) // 4) * 4
    padded_pwd = pwd_bytes.ljust(align_len, b'\x00')

    # Step 3: concatenation of padded_password + sha1_username
    combined = padded_pwd + sha1_username

    # Step 4: MD5 iterated 5 times
    result = combined
    for _ in range(5):
        md5 = hashlib.md5()
        md5.update(result)
        result = md5.digest()

    return result.hex()
```

## 5. BINARY TCP PROTOCOL (Device Server)

### Connection
Each device has its dedicated server (obtained from the device list):
devserver_ip:devserver_port   (e.g., 8.209.102.xxx:11812)
Do not use cloud.tenda.com.cn:11821 — that is the generic gateway.

### Encryption
Algorithm: AES/CBC/NoPadding
Key:        MQTTAESCBC128123  (16 bytes)
IV:         1234567890abcdef  (16 bytes)

### Session Key Derivation (EVP_BytesToKey)
```python
import hashlib

def evp_bytes_to_key(password: bytes, salt: bytes = b'tenda\x00\x00\x00') -> tuple:
    """Derives key (16 bytes) and IV (16 bytes) with SHA1, 5 rounds"""
    d = b''
    d_i = b''
    while len(d) < 32:
        d_i = hashlib.sha1(d_i + password + salt[:8]).digest()
        for _ in range(4):  # AES_ROUND = 5, so 4 extra iterations
            d_i = hashlib.sha1(d_i).digest()
        d += d_i
    return d[:16], d[16:32]  # key, iv
```

### Auth flow (CLOUD_ACCOUNT_AUTH)
State machine:
NOT_AUTH
→ send CMD_APP_ENCRYPT_Q (18)
← recv CMD_APP_ENCRYPT_A (19) + randNum
KEY_GENERATED
→ derive evp_key/iv from: VER_PSW + randNum + username + MD5(password)
→ send CMD_APP_CLOUD_AUTH_K (16) + TLV[username_enc] + TLV[MD5(key+iv+tenda)]
← recv CMD_APP_CLOUD_AUTH_L (17) + compress/encrypt flags
PUSH_REGISTE
→ send pushRegister(username, pushToken, "Android", language, autoLogin)
← recv success
AUTH_SUCCESS

### TLV Packet Format
struct TLV:
type:   uint8
length: uint16 (big-endian)
value:  bytes[length]

TLV Types:
TLV_NAME_TYPE    = 1   # encrypted username
TLV_AES_MD5_TYPE = 2   # MD5 of the AES hash


## 12. LOCAL CONNECTION (LOCAL_LINK)

### UDP Discovery
Send port:    11112  (broadcast on local network)
Receive port: 11111  (response from router)

UDP Packet Format (15 bytes):
```python
import struct
import hashlib

def build_mesh_udp_header() -> bytes:
    flag = 0xA0000000  # -168364040 signed

    # md5 of "Tenda" → collapsed into int32
    md5_bytes = hashlib.md5(b"Tenda").digest()
    md5_int = 0
    for i, b in enumerate(md5_bytes):
        md5_int += b << ((i % 4) * 8)
    md5_int &= 0xFFFFFFFF

    # CRC32 of the header (first without crc)
    header_no_crc = struct.pack(">II", flag, md5_int) + b'\x00\x00\x00\x00'
    crc = crc32_tenda(header_no_crc)

    # Final packet: flag(4) + md5(4) + crc(4) + [1,1,1](3)
    return struct.pack(">III", flag, md5_int, crc) + bytes([1, 1, 1])
```

### Local TCP Connection
Host: <gateway_ip>   (e.g., 192.168.0.1 — from DHCP)
Port: 9000
Same protocol as cloud: TLV + AES/CBC, same authentication state machine.

### Strategy for HA
For HA integration, the **Local Mode (HTTP)** described in Section 16 was chosen, as it offers the best balance between data completeness and ease of implementation without external dependencies.

## 14. SNEI_HEARTBEAT — LOCAL UDP BROADCAST (KEY DISCOVERY)

### Overview
Each mesh node broadcasts every ~5 seconds on UDP port 9801 with plain text status data — **no authentication required**.

### Packet Format
Multicast address: 01:80:c2:00:aa:16 (IEEE 802.1ag)
UDP src port:  9801
UDP dst port:  9801
Dst IP:        192.168.1.255 (broadcast subnet)

Binary header (64 bytes) + JSON payload

### Header Structure (64 bytes)
byte[0-5]:   dst MAC multicast (01:80:c2:00:aa:16)
byte[6-11]:  src MAC of the node
byte[12-13]: 6a6a (magic?)
byte[14-15]: 0010
byte[16-19]: variable
byte[20-21]: 8010
byte[22-23]: 0280 or 0284 or 0288
byte[24-29]: src MAC of the node (repeated)
byte[30-35]: ff:ff:ff:ff:ff:ff
byte[36-51]: "SNEI_HEARTBEAT" (ASCII)
byte[52-63]: padding \x00

### JSON payload — Key Fields
```json
{
  "is_mpp": 1,          // 1=master, 0=satellite
  "link_rate2mpp": 999999, // link rate to master (Mbps)
  "hop": 0,             // hop count from master (0=master)
  "ess": [              // configured SSIDs
    {"essid": "Tenda_Mesh_A1B2", "index": 0},
    {"essid": "Tenda_Mesh_A1B2_5G", "index": 1}
  ],
  "radio": [            // info for each radio
    {
      "work_band": 2,   // 2=2.4GHz, 4=5GHz
      "channel": 11,
      "load": 50,       // channel load %
      "sta_num": 11     // CONNECTED CLIENTS on this radio
    }
  ],
  "bss": [              // BSS info
    {
      "bssid": "CC:2D:21:XX:XX:B2",
      "sta_num": 11,
      "sta_max": 128,
      "radio": 1,
      "ess": 0,
      "ifname": "wlan0.2"
    }
  ]
}
```

- No cloud, no Tenda account
- Complete data (WAN, topology, client list, SSID) in a single call

## 17. Control API (setModules)

The `setModules` command is used to change settings or perform actions (like rebooting).

### Mesh / Node Reboot
Endpoint: `POST /;stok=<stok>/goform/setModules?modules=sys`

**Global Reboot Payload (Master + Satellites):**
```json
{
  "sys": {
    "reboot": 1
  }
}
```

**Node Reboot Payload (Specific Satellite):**
```json
{
  "sys": {
    "reboot": 1,
    "sn": "NODE_SN",
    "mac": "NODE_MAC_ADDR",
    "ip": "NODE_IP"
  }
}
```
*Note: Many Tenda firmwares close the TCP connection immediately after successfully receiving the reboot command (causing a `Server disconnected` error in the HTTP client), which should be interpreted as confirmation of the operation.*
